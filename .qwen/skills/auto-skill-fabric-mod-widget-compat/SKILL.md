При разработке модов для версий Minecraft 1.21+ (и выше) возникают проблемы с наследованием от стандартных виджетов (например, `ButtonWidget` или `PressableWidget`), так как многие методы рендеринга (например, `renderWidget`) стали `final`.

### Подход:

1. **Избегайте миксинов в стандартные виджеты:** Вместо того чтобы пытаться внедряться в `renderWidget` стандартных кнопок через `@Inject`, создавайте собственные виджеты.
2. **Наследование от `ClickableWidget`:** Используйте `ClickableWidget` как базовый класс. Он предоставляет достаточную гибкость для отрисовки (через метод `renderWidget`) и обработки событий нажатия (через `onClick`), не ограничивая вас `final` методами, характерными для специализированных кнопок.
3. **Инкапсуляция отрисовки:**
    - Реализуйте всю логику отрисовки внутри `renderWidget` вашего класса.
    - Используйте `DrawContext` для операций рисования.
    - Для реализации эффектов (бегущая строка, обрезка текста) используйте `context.enableScissor()` и `context.disableScissor()`, чтобы ограничить область отрисовки рамками вашего виджета.
4. **Обработка событий:** Переопределяйте `onClick(double mouseX, double mouseY)` для обработки нажатий, вместо поиска абстрактных методов `onPress()`.
5. **Совместимость с Gradle/Loom:** При возникновении проблем со сборкой (например, `archivesName` или `archivesBaseName`), используйте прямое обращение к переменным из `gradle.properties` (например, `archives_base_name`) в конфигурации `build.gradle` внутри блоков `base` и `jar`.
